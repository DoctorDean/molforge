# Security Policy

## Supported versions

| Version | Supported          |
| ------- | ------------------ |
| 0.x     | :white_check_mark: |

While `molforge` is pre-1.0, only the latest minor release receives security fixes.

## Reporting a vulnerability

**Please do not file a public issue.**

Report security vulnerabilities by emailing the maintainers (address to be added in `CODEOWNERS`) or by using GitHub's [private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing/privately-reporting-a-security-vulnerability) feature on this repository.

You should expect:

- An acknowledgement within 72 hours.
- A status update within 7 days.
- Coordinated disclosure once a fix is available.

Thank you for helping keep `molforge` and its users safe.
